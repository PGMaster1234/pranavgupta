Badminton Drone
July 2023
40-50 hours (this is for the entire project)
188 lines (this is for just the PID controller shown)
I coded this at UTD's (the University of Texas at Dallas) 8-week AI Summer Course, which I attended the summer following my Sophomore year. For my final project at the UTD course, I attempted to create a drone that could follow badminton shuttles, catch them, and retrieve them back to the user. This would require many steps, the first of which being detecting the shuttle. I started with collecting data by manually annotating 700 images of shuttles that I collected by sampling videos I recorded on my phone. I then used Roboflow's augmentation software to apply random rotations, recoloring, and noise to the images and was left with ~16,000 images. I then created a CNN model (Convolutional Neural Network) that could detect badminton shuttles within an image by using transfer learning to convert the YoloV8 object detection model to detect badminton shuttles in my image data. The next step was implementing a controller that would use the position data of the shuttle from the CNN model to move the drone. I couldn't directly enter lateral positions because the only parameter of the drone I could control was its acceleration by tilting the drone. The solution was a tuned PID controller. Using a custom bi-directional PID controller (shown in the background photo), I could move the drone towards the shuttle without major oscillations. Thefinal step was returning the drone back to the user. To do this, I simply recorded positon movements of the drone as it followed the shuttle and calculated a return path to reach the starting position. While I wasn't able to finish the physical drone in time for the presentation, the model and software required to create the drone were completed in time. Learn more about the project by looking at my final presentation here! Slides Presentation. The video shows the PID controller helping the drone (in white) follow the shuttle (in red). The frame rate is slowed down to 20fps so the tuning is visible.

First Reinforcement Learning Attempt
July 2023 - December 2023
20-30 hours
301 lines
This took about four attempts and a dozen files to complete. I was learning how to code discrete reinforcement learning using my own implementation of a neural network in a tile-based environment. The agent has to learn to reach the goal in a minimum number of steps while avoiding running into obstacles. Completing this project taught me about the Monte Carlo method, the SARSA algorithm(State_knot, Action, Reward, State_next, Action_next), and q-learning. Movie Sentiment
June 2023 - July 2023
5-10 hours
~600 lines total across 4 attempts
I coded this at UTD's (the University of Texas at Dallas) 8-week AI Summer Course, which I attended the summer following my Sophomore year. The problem was building a sentiment analysis LSTM model that could process text inputs (movie reviews) and say if they were positive or negative reviews. It used the IMDB dataset which has 50,00 data points, and it used a model that had ~70,000 parameters. I first attempted to solve this using my own from-scratch LSTM model but wasn't successful. Afterwards, I tried using Tensorflow but couldn't successfully implement it either. Finally, I decided to use PyTorch and got it to work. While limited by my computer's processing power during training, after the 15 epochs I was able to run (which took about half an hour) the model reached ~86% accuracy. This model had overfitting issues, though, because the training accuracy was reaching nearly 99%.
